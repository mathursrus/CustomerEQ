## Issues Found

### BUG-1: Admin sidebar not responsive on mobile [P0 — FIXED]

**Category:** UI/UX — Responsive Layout
**Severity:** Critical — makes entire admin area unusable on mobile
**Frequency:** 100% on screens < 768px

**Steps to reproduce:**
1. Navigate to any admin page (e.g., `/admin/campaigns`)
2. Resize browser to mobile width (375px)

**Expected:** Sidebar collapses or hides, content fills screen
**Actual:** Sidebar remained at full 256px width, overlapping content. Main content pushed off-screen.

**Fix applied:** Converted admin layout to client component with slide-out drawer pattern:
- Sidebar hidden off-screen (`-translate-x-full`) on mobile
- Hamburger button in header toggles sidebar open
- Dark overlay dismisses sidebar on tap
- Sidebar auto-closes on navigation
- Desktop behavior unchanged (`md:relative md:translate-x-0`)
- Added active nav link highlighting

**Evidence:**
- Before: `docs/evidence/ui-polish/admin-mobile-campaigns.png`
- After (closed): `docs/evidence/ui-polish/admin-mobile-fixed-closed.png`
- After (open): `docs/evidence/ui-polish/admin-mobile-fixed-open.png`

### BUG-2: "All Statuss" typo in filter dropdown [P1 — FIXED]

**Category:** UI/UX — Text/Copy
**Severity:** High — visible grammar error on Programs page
**Frequency:** 100% on Programs page

**Steps to reproduce:**
1. Navigate to `/admin/programs`
2. Look at the status filter dropdown

**Expected:** "All Statuses"
**Actual:** "All Statuss" (double s)

**Root cause:** `FilterBar` component appended `s` to all labels (`All ${filter.label}s`). When label is "Status", result is "Statuss".

**Fix applied:** Changed to `{filter.label}{filter.label.endsWith('s') ? 'es' : 's'}` — produces "All Statuses" and "All Types" correctly.

**Evidence:** `docs/evidence/ui-polish/programs-statuses-fixed.png`

---

### BUG-3: Missing favicon (404 error) [P2 — FIXED]

**Category:** UI/UX — Branding
**Severity:** Medium — 404 in console on every page load
**Frequency:** 100% on all pages

**Fix applied:** Added `apps/web/src/app/icon.svg` — indigo rounded square with "C" matching the sidebar logo. Next.js automatically serves it as the favicon.
