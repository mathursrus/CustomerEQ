Points & Thank You

The post-submit experience. Points are configured on your program, not the survey — this section displays what respondents will earn, and lets you customize the thank-you message.

Sourced from your program's earning rule for survey_completion. Points credit atomically via a LoyaltyEvent row when a response is accepted. Active campaigns may layer multipliers at runtime.

50 ⭐ Stars per accepted response — set in Acme Stars program
From program

V1 Per-survey override deferred — V0 keeps earning configuration on the program.

Shown after submit. Insert variables to reference the points the respondent earned.

Insert: {{points}} {{pointCurrencyName}} {{rewardLink}}

V0 variables only. {{programName}} and {{memberName}} aren't surfaced — program name is an internal concept, and member name isn't guaranteed to be on the Member record.

If set, post-submit redirects after 3 seconds. Embedded surveys ignore this and show the in-place thank-you.

ⓘ
Points are not shown on the survey form itself. Members see the points value in the invitation (email subject, embed snippet preview) and in the thank-you message via {{points}}. This protects response quality while still rewarding the member.
